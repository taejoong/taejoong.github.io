Research Questions
1.
Understanding the most challenging parts in deploying and managing DNSSEC correctly from the perspective of DNS administrators [DNSViz, OpenINTEL]
2.
How can we help the DNS administrators deploy and configure DNSSEC correctly? [DNSViz + ML] (before deploying it)
3.
Can we come up with some "radical" approach to solve some really-hard-to-solve DNSSEC's challenges?

Playing with DNSViz dataset
For a given domain, here is the raw (i.e., "dnsviz probe" output) information for the latest analysis: https://dnsviz.net/d/mitre.org/REST/raw/ Same, but for a specific analysis (specific date/time): https://dnsviz.net/d/mitre.org/XuPYVA/REST/raw/ The XuPYVA is the encoded timestamp, generated from this code (datetime to URL and vice-versa): https://github.com/dnsviz/dnsvizwww/blob/master/dnsvizwww/util.py#L45 https://github.com/dnsviz/dnsvizwww/blob/master/dnsvizwww/util.py#L49 This URL gives the warnings associated with the most current analysis (i.e., a "processed", "not raw" analysis): https://dnsviz.net/d/mitre.org/REST/processed/?l=warning Same, but for a specific analysis (specific date/time): https://dnsviz.net/d/mitre.org/XuPYVA/REST/processed/?l=warning To navigate, you can use the "meta" url: https://dnsviz.net/d/mitre.org/REST/meta/ I don't have the URLs for the timestamps in the meta information. With the code above it should be trivial, but just stating it for completeness.
2. WHOIS Records
We need mappings between a domain and its registrar. Because registrars usually impose a rate limit for WHOIS queries (also we do not need any registrant information), our goal is to send WHOIS record requests only to registries (e.g., Verisign). @Spencer Roth is working on this project: Scanning WHOIS Records (led by @Spencer Roth)
For .NL domains, we might be able to get some help from SIDN (Moritz)
3. OpenIntel
DNS scans based on TLD zones; for example, .com scans span from March 1st, 2015 [1]
4. CTLogs
Tijay has been collecting all domains appeared on certificates registered on CTLogs since July, 2020 – However, it contains only A, CAA records; do not have any DNSSEC-relevant data. Tijay can modify the code to, at least, store (1) whether the domain is signed or unsigned, (2) valid or not.
5. Farsight Passive DNS datasets
Casey is currently allowed to send 5,000 queries to Farsight DB (he will be working on the increase the limit) – We confirmed that historical DNS records are available;
•
Some domain's DNSSEC relavant records are collected even more than 10 years (e.g., 2010)
•
Comparing to OpenIntel that only covers SLDs, Farsight can cover any domains
•
One minor challenge is that they do not come with RRSIG when you request a DNS record: you have to explicitly request "RRSIG" – Tijay's question; can we specify the base RRType of RRSIG? in other words, the RRSIG can be a "A" record's RRSIG or "MX" record's RRSIG
This dataset will be potentially very useful for those of DNS records that OpenIntel does not cover.
